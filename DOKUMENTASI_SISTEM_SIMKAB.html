BAB 1: Pendahuluan & Deskripsi Sistem

1.1 Latar Belakang Kehadiran SIMKAB

Dalam industri lembaga keuangan/perbankan, presisi pengelolaan administrasi personalia bersifat krusial. Permasalahan absensi fiktif, ketidaksesuaian penggajian dengan jabatan struktural, lambatnya pengajuan cuti, serta hilangnya berkas mutasi fisik merupakan masalah klasik yang menurunkan efektivitas bisnis bank. SIMKAB hadir sebagai solusi otomatisasi tata kelola personalia bank yang aman, transparan, dan real-time.

1.2 Tujuan Utama Proyek

Sistem ini dirancang untuk mencapai standardisasi manajemen data karyawan secara mandiri (Self-Service) dan otomatisasi bisnis dalam satu pintu.

Akurasi Presisi Kehadiran Harian

Memastikan pencatatan kehadiran karyawan bank dilakukan di lokasi riil kantor menggunakan deteksi koordinat GPS dan validasi selfie wajah anti-fraud.

Otomatisasi Skala Gaji & Tunjangan

Mengintegrasikan database jabatan baku secara otomatis dengan nominal penggajian pokok dan tunjangan tanpa intervensi manual (menghindari manipulasi).

Rekam Jejak Karir (Audit Trail)

Melacak dengan detail riwayat perjalanan karir karyawan dari penempatan pertama, mutasi divisi, promosi kenaikan pangkat, hingga penyesuaian gaji terkait.

BAB 2: Pembagian Peran & Hak Akses (Role & Otorisasi)

Aplikasi SIMKAB menggunakan metode keamanan RBAC (Role-Based Access Control). Hak istimewa sistem dipisah secara tegas berdasarkan fungsionalitas kerja di dunia nyata:

Informasi Keamanan Sesi

Otorisasi ini divalidasi ganda. Selain pengecekan tampilan di JavaScript (Frontend), berkas api.php di sisi server divalidasi menggunakan pemeriksaan variabel $_SESSION['role'] sebelum memproses query database sensitif.

BAB 3: Arsitektur Sistem & Struktur Berkas

Sistem SIMKAB mengadopsi model Modular Single-Page Application (SPA). Konsep ini memuat seluruh antarmuka pada satu halaman fisik induk, sedangkan navigasi sub-tab dimuat secara instan melalui Vanilla JavaScript dinamis tanpa pemuatan ulang (refresh) halaman secara penuh.

3.1 File Utama & Struktur Hubungan

1
index.php (Shell Induk SPA)

Berperan sebagai wadah utama halaman. Memuat sidebar navigasi perbankan, memanggil berkas CSS, pustaka JS, dan merender potongan menu modular secara asinkron.

2
api.php (Gateway Kontrol Tunggal)

Mengontrol seluruh pertukaran data ke database. Menerima request AJAX Fetch dari browser dan menyuplai respon balik terstruktur dalam format JSON.

3
assets/js/app.js (Logika Frontend)

Mengendalikan interaksi pengguna, penghitungan jarak GPS Geofencing, pemindaian kamera selfie, konversi gambar ke Base64, pengelolaan state UI, serta eksekusi cetak slip PDF.

4
modules/ (Modul Halaman Parsial)

Berisi 10 berkas tampilan antarmuka terpisah (dashboard.php, karyawan.php, mutasi.php, dsb.) yang disisipkan dinamis ke dalam halaman utama.

BAB 4: Struktur Basis Data & Kamus Data Fisik

Basis data relasional menggunakan nama db_simkab yang terdiri dari 12 tabel terintegrasi dengan status integritas kunci asing (*foreign key*) bermode ON DELETE CASCADE pada data master pegawai karyawan(id).

1. Tabel: karyawan

Menyimpan data master identitas diri, divisi kerja, jabatan aktif, gaji dinamis hasil sinkronisasi jabatan standar, dan sisa jatah cuti karyawan bank.

divisi VARCHAR(50) - Nama divisi kerja penempatan aktif
jabatan VARCHAR(100) - Jabatan struktural resmi pegawai
gaji_pokok DECIMAL(15,2) - Gaji pokok bulanan (auto-sync jabatan)

Tabel acuan tunggal skala gaji dan tunjangan untuk 17 tingkatan jabatan perbankan yang dibagi rata pada 4 Divisi utama bank.

Nama Field Tipe Data Atribut Kunci Keterangan Deskripsi

Fitur E-Absensi: GPS Geofencing & Haversine Formula

Sistem E-Absensi SIMKAB mewajibkan verifikasi lokasi fisik karyawan bank sebelum melakukan presensi harian untuk menghindari pemalsuan koordinat.

Batas Koordinat Geofencing Kantor:

  • Latitude Acuan (Kantor Pusat): -4.0113509
  • Longitude Acuan (Kantor Pusat): 122.5177928
  • Radius Toleransi Maksimum: 500 Meter
Algoritma Haversine (Penyusunan Jarak Geografis)
d = 2R · arcsin( √( sin²(Δlat/2) + cos(lat₁) · cos(lat₂) · sin²(Δlon/2) ) )

JavaScript menghitung jarak linear terpendek d bumi bulat secara real-time. Jika d <= 500 meter, status absensi menjadi Dalam Jangkauan dan karyawan dipersilakan melakukan Check-In. Jika melebihi batas, tombol terkunci otomatis.

Aturan Batas Waktu & Keterlambatan Absensi (Lateness Rules):

Sistem backend secara otomatis memproses status kehadiran karyawan saat *Check In* berdasarkan zona waktu setempat (WITA / Asia/Makassar) dengan pembagian toleransi berikut:

Kategori Waktu Check-In Status Kehadiran Keterangan / Tindakan
00:00:00 s.d 08:00:00 pagi Hadir Tepat waktu, tidak dikenakan denda atau sanksi keterlambatan.
08:00:01 s.d 09:00:00 pagi Terlambat Dikenakan status terlambat dan denda potongan sesuai kebijakan HRD.
Di atas 09:00:00 pagi Tidak Hadir Dianggap Alfa / Mangkir kerja, otomatis dicatat merah sebagai tidak hadir.

Fitur Cuti Karyawan: Kuota Cuti Otomatis

Hak sisa jatah libur cuti tahunan dikelola secara ketat dan diproses otomatis oleh database untuk meminimalisasi ketidaksesuaian administrasi.

A
Pengajuan Durasi Hari

Sistem menghitung durasi: (Tanggal Selesai - Tanggal Mulai) + 1. Jika durasi melebihi kuota sisa_cuti karyawan, form ditolak otomatis di sisi klien.

B
HRD Otorisasi / Approval

HRD masuk ke dasbor dan memverifikasi pengajuan cuti berstatus Pending. HRD berhak menyetujui atau menolak permohonan tersebut.

C
Pemotongan Saldo Cuti di Database

Jika disetujui, backend memotong kuota: sisa_cuti = sisa_cuti - durasi_hari di tabel karyawan lewat transaksi aman database SQL.

Fitur Penggajian: Formulasi Slip Gaji (Take Home Pay)

Kalkulasi gaji bersih bulanan karyawan perbankan ditarik secara dinamis dari profil karyawan dan disintesis dengan variabel denda absensi serta bonus kinerja.

Formulasi Take Home Pay (Gaji Bersih)
Take Home Pay = Gaji Pokok + Tunjangan Jabatan + Bonus Kinerja - Denda Potongan

Rincian penghasilan ini tersimpan permanen di tabel payroll bulanan. Pegawai yang bersangkutan dapat mengakses sub-tab Payroll untuk mengunduh berkas fisik slip gaji digital berformat PDF eksklusif secara instan.

Fitur Mutasi & Rotasi: Penyesuaian Jabatan & Gaji Otomatis

Guna menghindari manipulasi nominal penggajian secara manual oleh oknum tidak bertanggung jawab, sistem SIMKAB menerapkan pembaruan gaji dinamis berbasis kueri standar jabatan saat mutasi terjadi.

Mekanisme Sinkronisasi Gaji Mutasi

Ketika HRD mengajukan rotasi pegawai ke divisi baru dan jabatan struktural baru melalui form mutasi, sistem backend api.php akan memicu kueri pencarian gaji standar untuk pasangan divisi dan jabatan baru tersebut di tabel standar_jabatan.

Setelah mendapatkan data tersebut, sistem melakukan query UPDATE secara atomik dalam satu transaksi untuk memperbarui kolom divisi, jabatan, gaji_pokok, dan tunjangan karyawan tersebut di tabel utama karyawan secara instan.
